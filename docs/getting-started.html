Getting Started with MCP Agentify v2.0

MCP Agentify v2.0 is an enhanced AI-powered platform with modular architecture that transforms your existing REST APIs into Model Context Protocol (MCP) servers. This guide covers the new features including AI integration, comprehensive debugging tools, and enterprise-grade health monitoring.

Installation

CLI Tool

Install the CLI tool globally using npm:

# Install globally
npm install -g mcp-agentify

# Verify installation
agentify --version

# Or use with npx (no installation required)
npx mcp-agentify --help

VS Code Extension

Install the VS Code extension for a visual interface:

# Install from command line
code --install-extension saurabhkohli.mcp-agentify

# Or search "MCP Agentify" in VS Code Extensions marketplace

Quick Start

1. Analyze Your Project

First, analyze your existing API project to detect endpoints:

# Analyze current directory
agentify analyze

# Analyze specific project
agentify analyze ./my-api-project

# Get detailed JSON output
agentify analyze --json > analysis.json

2. Generate MCP Server

Generate a complete MCP server from your API:

# Generate with default settings
agentify generate

# Generate with custom output directory
agentify generate --output ./my-mcp-server

# Generate with specific options
agentify generate --format typescript --tests --docs

3. Interactive Setup

Use the interactive wizard for guided setup:

# Launch interactive setup
agentify interactive

Configuration

Create Configuration File

# Initialize configuration
agentify init

# This creates agentify.config.json with default settings

Configuration Options

{
  "serverName": "my-mcp-server",
  "description": "Generated MCP server",
  "version": "1.0.0",
  "outputFormat": "typescript",
  "includeTests": true,
  "includeDocumentation": true,
  "excludeEndpoints": [],
  "transformRules": [],
  "security": {
    "enableJWT": false,
    "enableRBAC": false,
    "enableValidation": true

Examples

Express.js API

For an Express.js application with the following structure:

// app.js
app.get('/users', getUsersHandler);
app.post('/users', createUserHandler);
app.get('/users/:id', getUserByIdHandler);
app.put('/users/:id', updateUserHandler);
app.delete('/users/:id', deleteUserHandler);

Run the analysis and generation:

# Analyze the Express app
agentify analyze ./express-app

# Generate MCP server
agentify generate ./express-app --output ./user-mcp-server

FastAPI Application

# main.py
@app.get("/items/")
async def read_items():
    return items

@app.post("/items/")
async def create_item(item: Item):
    return item
# Generate from FastAPI
agentify generate ./fastapi-app --format typescript

OpenAPI Specification

# Generate from OpenAPI/Swagger file
agentify generate ./swagger.json --type openapi
agentify generate ./openapi.yaml --type openapi

VS Code Extension Usage

Getting Started with VS Code Extension

  1. Open your API project in VS Code
  2. Open Command Palette (Ctrl+Shift+P / Cmd+Shift+P)
  3. Run "MCP Agentify: Analyze Project"
  4. View results in the MCP Agentify sidebar
  5. Generate server using "MCP Agentify: Generate MCP Server"

  • One-Click Generation: Generate servers with visual configuration

Troubleshooting

Common Issues

No Endpoints Detected

  • Ensure your project uses a supported framework
  • Check that route files are in standard locations
  • Run analysis from the project root directory
  • Use the --verbose flag for detailed logging

Generation Fails

  • Check that the output directory is writable
  • Ensure Node.js version 18+ is installed
  • Verify the analysis step completed successfully
  • Check for any dependency conflicts

VS Code Extension Issues

  • Ensure the CLI tool is installed: npm install -g mcp-agentify
  • Check VS Code settings for correct CLI path
  • Reload VS Code after installing the extension
  • Check the Output panel for error messages

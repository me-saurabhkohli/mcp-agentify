agentify analyze

Analyzes a project to detect API endpoints and framework type.

Usage

agentify analyze [project-path] [options]

Options

Option Description Default
-p, --project <path> Path to the project directory .
-t, --type <type> Project type: rest-api, nodejs, openapi auto
--json Output analysis results as JSON false
--verbose Enable verbose logging false

Examples

# Analyze current directory
agentify analyze

# Analyze specific project with JSON output
agentify analyze ./my-api --json

# Analyze OpenAPI specification
agentify analyze ./swagger.json --type openapi

agentify generate

Generates a complete MCP server from analyzed project.

Usage

agentify generate [project-path] [options]

Options

Option Description Default
-p, --project <path> Path to the project directory .
-o, --output <path> Output directory for generated MCP server ./mcp-server
--format <format> Output format: typescript, javascript typescript
--tests Include test files true
--docs Include documentation true
--security Enable security features false
--plugins <list> Comma-separated list of plugins ""
--dry-run Show what would be generated without creating files false

Examples

# Generate with default settings
agentify generate

# Generate with custom output and format
agentify generate --output ./my-mcp-server --format javascript

# Generate with security and plugins
agentify generate --security --plugins rest-analyzer,template-engine

# Dry run to preview generation
agentify generate --dry-run

agentify interactive

Launches an interactive setup wizard with guided configuration.

Usage

agentify interactive [project-path]

Features

  • Automatic project detection
  • Step-by-step configuration
  • Smart default suggestions
  • Configuration validation
  • Progress tracking

Example

# Launch interactive setup
agentify interactive

# Launch for specific project
agentify interactive ./my-api-project

agentify init

Initializes a configuration file with default settings.

Usage

agentify init [options]

Options

Option Description Default
-o, --output <path> Configuration file output path ./agentify.config.json
--template <name> Configuration template: basic, advanced, enterprise basic

Example

# Create basic configuration
agentify init

# Create advanced configuration
agentify init --template advanced

Configuration File

The agentify.config.json file allows you to customize the generation process:

{
  "serverName": "my-mcp-server",
  "description": "Generated MCP server",
  "version": "1.0.0",
  "outputFormat": "typescript",
  "includeTests": true,
  "includeDocumentation": true,
  "excludeEndpoints": [
    "GET:/health",
    "POST:/internal/*"
  ],
  "transformRules": [
    {
      "pattern": "get_user_by_id",
      "replacement": "getUser",
      "type": "endpoint"
    }
  ],
  "security": {
    "enableJWT": true,
    "enableRBAC": true,
    "enableValidation": true,
    "enableAuditLogging": false
  },
  "plugins": [
    "rest-analyzer",
    "template-engine",
    "security"
  ],
  "customTemplates": "./templates",
  "environment": {
    "development": {
      "outputPath": "./dev-mcp-server"
    },
    "production": {
      "outputPath": "./prod-mcp-server",
      "security": {
        "enableJWT": true,
        "enableRBAC": true,
        "enableAuditLogging": true
      }
    }
  }
}

Environment Variables

MCP Agentify supports configuration via environment variables:

Variable Description Example
AGENTIFY_CONFIG Path to configuration file ./config/agentify.json
AGENTIFY_OUTPUT Default output directory ./generated-servers
AGENTIFY_FORMAT Default output format typescript
AGENTIFY_VERBOSE Enable verbose logging true
